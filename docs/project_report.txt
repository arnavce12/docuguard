# DocuGuard - Comprehensive Project Report

## 1. Project Overview
DocuGuard is a state-of-the-art document analysis and fraud detection platform. It leverages advanced AI (Google Gemini) to provide users with deep insights into document authenticity, health, and classification.

### Core Value Propositions:
- **Fraud Detection**: Identifying potential forgeries, inconsistencies, and tampering in documents.
- **Document Health Score**: Evaluating document completeness, legibility, and validity for legitimate files.
- **Know Your Document (KYD)**: Automatically identifying document types, issuing authorities, and key data points.
- **Unified History**: Securely tracking and managing all document analyses.

## 5. Cybersecurity Optimizations & Best Practices

DocuGuard implements multiple layers of security to protect user data and ensure system stability:

### 1. Authentication & Authorization
- **Supabase JWT**: All sensitive endpoints require a valid Bearer token.
- **User Isolation**: Backend logic strictly verifies that a user can only access/modify their own records using the `user_id` extracted from the JWT.
- **Supabase RLS**: Row-Level Security is enabled on all database tables to prevent unauthorized data access at the database level.

### 2. Request Protection
- **Rate Limiting**: Implemented via `SlowAPI` on every functional endpoint to prevent automated abuse and DoS attacks.
- **CORS Protection**: Strict `CORSMiddleware` configuration restricted to specific authorized frontend domains.
- **Input Validation**: Pydantic models used for strict request body validation.

### 3. File & Path Security
- **Signed URLs**: Private files in Supabase Storage are NEVER exposed via public URLs. Temporary signed URLs are generated on-the-fly after backend validation.
- **Path Traversal Protection**: Explicit checks in the `/api/signed-url` endpoint to block `..` or `\` characters and ensure the path starts with the user's own `user_id`.
- **Public/Private Separation**: Clear separation between public demo assets and sensitive user documents.

### 4. Infrastructure Security
- **Serverless Architecture**: Leveraging Vercel's serverless functions reduces the attack surface compared to long-running traditional servers.
- **Environment Variables**: Sensitive keys (Supabase, Gemini) are managed through secure environment variables, never committed to source control.

## 8. Database & Storage Schema

The system uses Supabase (PostgreSQL) for structured data and Supabase Storage for binary files.

### Database Tables:
- **`scans`**:
    - `id`: UUID (Primary Key)
    - `user_id`: UUID (Foreign Key to Auth.Users)
    - `fraud_likelihood`: Text (low, medium, high)
    - `confidence_score`: Integer (0–100)
    - `red_flags`: JSONB (Array of detected issues)
    - `explanation`: Text (AI-generated reasoning)
    - `document_label`: Text (User-defined or AI-suggested name)
    - `health_score_overall`: Integer (0–100)
    - `health_score_grade`: Text (A, B, C, D, F)
    - `health_score_axes`: JSONB (Detailed metrics)
    - `health_score_flags`: JSONB (Health-specific issues)
    - `storage_path`: Text (Path in Supabase Storage)
    - `is_public_demo`: Boolean
    - `created_at`: Timestamp

- **`kyd_analyses`**:
    - `id`: UUID (Primary Key)
    - `user_id`: UUID (Foreign Key)
    - `document_type`: Text
    - `common_name`: Text
    - `issuing_authority`: Text
    - `purpose`: Text
    - `typical_use_cases`: JSONB
    - `key_fields_present`: JSONB
    - `data_categories`: JSONB
    - `notes`: Text
    - `file_name`: Text (Storage path)
    - `created_at`: Timestamp

### Storage:
- **Bucket**: `documents`
- **Structure**: `uploads/` (User uploads) and specific user folders for isolation.
